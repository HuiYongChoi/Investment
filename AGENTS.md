Investment Navigator

Investment Navigator는 한국 상장기업의 가치투자 판단을 돕는 웹 대시보드입니다.
앞으로의 기본 시세/차트 소스는 아래 두 가지를 기준으로 가정합니다.

- `DART`: 공시, 재무제표, 최근 3년 사업/반기/분기 보고서
- `Python yfinance (Yahoo Finance)`: 현재가, 일봉/주봉, 제한적 분봉, 기술적 분석용 OHLCV

`KRX Open API`는 보조 검증용으로 둘 수 있으나, 실시간 시세와 차트의 주 소스로 가정하지 않습니다.
기존 저장소에 남아 있는 `Kiwoom REST API` 또는 `KRX` 중심 구현은 레거시로 간주하고, 새 작업은 `Python yfinance` 우선으로 설계합니다.

## 제품 기준

1. 기업 검색
- 검색 드롭다운은 반드시 `종목명 · 종목코드` 한 줄 형식으로 표기합니다.
- 종목명, 별칭, 6자리 종목코드로 검색 가능해야 합니다.
- 6자리 종목코드는 시장에 따라 Yahoo Finance 티커로 변환할 수 있어야 합니다.
- `KOSPI -> .KS`, `KOSDAQ -> .KQ` 규칙을 기본값으로 사용합니다.

2. 주가 정보
- 상단 가격 스트립에는 `현재가`, `전일대비`, `등락률`, `시가`, `고가`, `저가`, `거래량`을 표시합니다.
- 현재가와 차트 소스는 항상 화면에 명확히 표시합니다.
- `Python yfinance`를 사용한 경우 UI에 `Yahoo Finance (yfinance Python)`처럼 출처를 오해 없이 표기합니다.
- API 실패 시 가짜 가격을 그려 넣지 말고, `실시간 시세 미연결` 상태를 솔직하게 표기합니다.

3. 차트
- 가격 차트는 `Python yfinance`로 수집한 OHLCV를 기준으로 구성합니다.
- 기본 표시는 `1M`, `3M`, `주봉`, `YTD`를 지원합니다.
- 캔들 툴팁에는 `날짜`, `시가`, `고가`, `저가`, `종가`, `등락률`이 보여야 합니다.
- 매수/매도 시그널은 가격 차트가 아니라 기술적 분석 차트에 표시합니다.
- 분봉은 yfinance 제공 범위 안에서만 사용하고, 지원 범위를 넘는 경우 빈 상태와 명확한 안내를 우선합니다.

4. 재무/공시
- 최근 3개년 연간 실적과 최신 분기 실적을 함께 보여줍니다.
- 최근 보고서 목록은 최신 공시 순으로 `사업보고서`, `반기보고서`, `분기보고서`를 정리합니다.
- 최신 분기 판단은 DART 실제 공시 기준으로 처리합니다.

5. 기술적 분석
- `RSI`, `MACD`, `스토캐스틱`, `볼린저 밴드`, `이동평균선`을 지원합니다.
- 보조지표별 매수/매도/중립 개수를 집계해 최종 의견을 보여줍니다.
- 보조지표 계산의 입력 OHLCV도 `Python yfinance` 정규화 데이터를 기준으로 맞춥니다.

6. 부가 기능
- `Kakao 로그인`을 지원합니다.
- `Google AI Studio (Gemini)` 기반 브리핑을 제공하되, 실패 시 로컬 브리핑으로 안전하게 대체합니다.
- 상단 바에는 `현재 시각`, `USD/KRW`, `JPY/KRW`, `금 시세`를 표시합니다.

## 구현 원칙

- 비밀값은 코드에 하드코딩하지 않습니다. 환경변수 또는 서버 외부 비밀파일에서 읽습니다.
- `yfinance`는 가능하면 브라우저 직접 호출이 아니라 Python 계층에서 수집/정규화한 뒤 사용합니다.
- Yahoo Finance 티커 변환 규칙은 코드 한 곳에서 일관되게 관리합니다.
- yfinance 시세가 없을 때는 부정확한 합성 차트보다 `빈 상태 + 명확한 오류 메시지`를 우선합니다.
- 차트/가격/브리핑의 데이터 출처는 UI에서 오해 없이 보이도록 유지합니다.
- 새 구현이나 리팩터링은 `date/open/high/low/close/volume` 형태의 정규화된 JSON 계약을 우선합니다.
